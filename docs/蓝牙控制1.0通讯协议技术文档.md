# 蓝牙控制 1.0 通讯协议技术文档

> 版本：V1.6  
> 更新日期：2026-07-10  
> 适用范围：Hanami Web Serial 蓝牙控制面板 ↔ 蓝牙 IC 芯片（BLE GATT）

---

## 1. 概述

本文档定义 Web 蓝牙控制面板（主机，Host）与蓝牙 IC 芯片（从机，Chip）之间的二进制通讯协议。通信基于 **BLE GATT** 特性（Characteristic）进行双向数据交换，采用定长帧 + 累加和校验（CRC）的方式确保数据完整性。

### 1.1 通信模型

```
┌─────────────────┐         BLE GATT          ┌─────────────────┐
│  Web 蓝牙控制面板  │ ◄──────────────────────► │   蓝牙 IC 芯片    │
│  (Host / 主机)    │   Write / Notify         │  (Chip / 从机)    │
└─────────────────┘                          └─────────────────┘
```

- **主机 → 芯片**：通过 GATT Write 发送控制指令，帧头 `0xAA`
- **芯片 → 主机**：通过 GATT Notify 回复状态数据，帧头 `0x55`

### 1.2 通信参数

| 参数 | 值 |
|------|-----|
| 传输层 | BLE GATT（Bluetooth Low Energy） |
| 帧头（主机→芯片） | `0xAA` |
| 帧头（芯片→主机） | `0x55` |
| Web 标识 | `0x01` |
| 校验方式 | 累加和 CRC（sum modulo 256） |
| 字节序 | 大端序（Big Endian） |

## 2. 帧格式（Frame Format）

### 2.1 帧结构

每一帧由以下字段按顺序组成：

```
┌──────────┬──────────┬──────────┬──────────┬─────────────────┬──────────┐
│ Frame Hdr│  Web ID  │   LEN    │   CMD    │    VAL0...VALn   │   CRC    │
│  1 byte  │  1 byte  │  1 byte  │  1 byte  │    n bytes       │  1 byte  │
│ 0xAA/0x55│  0x01    │  可变     │  可变     │   可变            │  可变     │
└──────────┴──────────┴──────────┴──────────┴─────────────────┴──────────┘
```

| 偏移 | 字段名 | 字节数 | 说明 |
|------|--------|--------|------|
| 0 | Frame Header | 1 | 帧头标识：主机发送 `0xAA`，芯片回复 `0x55` |
| 1 | Web ID | 1 | 固定为 `0x01`，标识 Web 端通信 |
| 2 | LEN | 1 | 数据长度 = 1（CMD）+ 参数 VAL0..VALn 的字节数 |
| 3 | CMD | 1 | 命令码，标识具体功能（详见第 3 章命令体系） |
| 4...4+n-1 | VAL0...VALn | n | 参数数据，长度和含义取决于具体 CMD |
| 4+n | CRC | 1 | 累加和校验值，覆盖从 Frame Header 到最后一个 VAL 字节的所有字节 |

### 2.2 LEN 长度计算

```
LEN = 1 (CMD 自身) + 参数字节数
```

示例：
- 无参数命令（如 `SYS_ASK`）：LEN = 1 + 0 = 1
- 带 1 字节参数（如 `EQ_VOL_VAL`）：LEN = 1 + 2 = 3（VAL0=开关, VAL1=值）
- 10 段 EQ（`EQ_VOL_FRE_VAL`）：LEN = 1 + 11 = 12（VAL0=开关, VAL1-10=10 段值）

### 2.3 最小帧长度

最小帧为 5 字节：Frame Header (1) + Web ID (1) + LEN (1) + CMD (1) + CRC (1)。当 LEN = 1（只有 CMD，无参数）时适用。

## 3. CRC 校验

### 3.1 算法

```
CRC = (FrameHeader + WebID + LEN + CMD + VAL0 + ... + VALn) mod 256
```

即：将帧中 CRC 字段之前的**所有字节**累加，取结果的低 8 位。

### 3.2 JavaScript 实现

```javascript
function calculateCRC(bytes) {
    let crc = 0x00;
    for (let i = 0; i < bytes.length; i++) {
        crc += bytes[i];
    }
    return crc & 0xFF;
}
```

### 3.3 计算示例

**发送音乐主音量设置命令（CMD=0x11，开启，音量=22）：**

```
帧内容: AA 01 03 11 01 16
累加:   0xAA + 0x01 + 0x03 + 0x11 + 0x01 + 0x16
      = 170 + 1 + 3 + 17 + 1 + 22
      = 214
      = 0xD6
CRC = 0xD6
完整帧: AA 01 03 11 01 16 D6
```

**芯片回复确认：**

```
帧内容: 55 01 03 11 01 16
累加:   0x55 + 0x01 + 0x03 + 0x11 + 0x01 + 0x16
      = 85 + 1 + 3 + 17 + 1 + 22
      = 129
      = 0x81
CRC = 0x81
完整帧: 55 01 03 11 01 16 81
```

## 4. 命令体系

命令码（CMD）按功能分组，每组对应一个功能 Map 字节（byte），每个功能占该字节中的一个 bit 位。

### 4.1 系统功能（System）

**Map Byte 0（bit 0-7）**

| CMD | Bit | 枚举名 | 中文描述 | VAL 范围 | 读写 | 说明 |
|-----|-----|--------|----------|----------|------|------|
| `0x00` | 0 | `SYS_ASK` | 问答机制 / 询问是否在线 | 无 | R | 芯片回复一包数据 |
| `0x01` | 1 | `SYS_CHIP_SUPPORT_MAP` | 询问芯片支持功能表 | 无 | R | 初次连接询问 |
| `0x02` | 2 | `SYS_ASK_ALL` | 请求芯片发送全部支持的 CMD | 无 | R | 芯片队列回复多包数据（不常用） |
| `0x03` | 3 | `SYS_ASK_CMD` | 请求芯片发送指定 CMD 的功能数值 | VAL0 = 目标 CMD (0-255) | R | 常用 |
| `0x04` | 4 | `SYS_ASK_MAP` | 请求芯片按 bit map 发送所有为 1 的功能 | VAL0..n（与 map 表字节数相关） | R | 初次连接 / 页面切换 |
| `0x05` | 5 | `SYS_ASK_BAT` | 询问电量 | 无 | R | 初次连接 / 每 10s 轮询 |
| `0x06` | 6 | `SYS_ASK_MCU_ID` | 询问芯片 ID | 无 | R | 设备信息 |
| `0x07` | 7 | `SYS_ASK_MID` | 询问制造商识别码（MID） | 无 | R | 设备信息 |

## 5. 芯片回复格式

芯片收到命令后，通过 BLE GATT Notify 回复确认帧，帧头为 `0x55`。以下为各命令的典型回复帧示例。

### 5.1 系统功能回复

| 命令 | 回复帧（十六进制） | 解析 |
|------|-------------------|------|
| SYS_ASK (`0x00`) | `55 01 02 00 01 59` | VAL0=`0x01`（在线确认） |
| SYS_CHIP_SUPPORT_MAP (`0x01`) | `55 01 0F 01 FF 00 3F 0F 9F 7F 80 FF FF 01 87 FF 00 8F 65` | 15 字节功能 Map 表 |
| SYS_ASK_BAT (`0x05`) | `55 01 02 05 40 9D` | 电量 = `0x40` = 64% |
| SYS_ASK_MCU_ID (`0x06`) | `55 01 07 06 88 88 88 88 88 88 93` | 芯片 ID = `88 88 88 88 88 88` |
| SYS_ASK_MID (`0x07`) | `55 01 05 07 01 26 05 31 BF` | MID = `01 26 05 31` |

### 5.2 音效功能回复

| 命令 | 回复帧（十六进制） | 解析 |
|------|-------------------|------|
| EQ_VOL_RESET (`0x10`) | `55 01 02 10 01 69` | VAL0=`0x01`（确认） |
| EQ_VOL_VAL (`0x11`) | `55 01 03 11 01 16 81` | VAL0=`0x01`(开), VAL1=`0x16`=22 |

## 6. 功能 Map 表机制

### 6.1 概述

芯片通过**功能 Map 表（Capability Bit Map）**向主机报告其支持的功能集。Map 表的每个 bit 对应一个 CMD，bit=1 表示该功能已实现。

### 6.2 Map 表结构

Map 表字节与 CMD 的对应关系：

| Map Byte | Bit 范围 | 对应 CMD 范围 | 功能组 |
|----------|---------|--------------|--------|
| byte 0 | 0-7 | `0x00`-`0x07` | 系统功能 |
| byte 1 | 8-15 | `0x08`-`0x0F` | 系统功能（预留） |
| byte 2 | 16-23 | `0x10`-`0x17` | 音效功能 — 音乐 EQ |
| byte 3 | 24-31 | `0x18`-`0x1F` | 音效功能 — 增强音效 |
| byte 4 | 32-39 | `0x20`-`0x27` | 音效功能 — 播放控制 |
| byte 5 | 40-47 | `0x28`-`0x2F` | MIC 功能 |
| byte 6 | 48-55 | `0x30`-`0x37` | MIC 功能（预留 + 保存） |
| byte 7 | 56-63 | `0x38`-`0x3F` | 灯光功能（模式 0-6） |
| byte 8 | 64-71 | `0x40`-`0x47` | 灯光功能（模式 7-14） |
| byte 9 | 72-79 | `0x48`-`0x4F` | 灯光功能（模式 15 + 预留） |
| byte 10 | 80-87 | `0x50`-`0x57` | 灯光功能（参数 + 保存） |
| byte 11 | 88-95 | `0x58`-`0x5F` | 文字功能（内容 + 模式 0-6） |
| byte 12 | 96-103 | `0x60`-`0x67` | 文字功能（预留） |
| byte 13 | 104-111 | `0x68`-`0x6F` | 文字功能（样式 + 保存） |
| byte 14+ | 112+ | `0x70`+ | 扩展保留 |

### 6.3 查询流程

1. **主机发送 `SYS_CHIP_SUPPORT_MAP`（CMD `0x01`）** — 芯片回复完整的 15 字节 Map 表
2. **主机发送 `SYS_ASK_MAP`（CMD `0x04`）** — 附带按 bit 设置的目标 Map，芯片仅回复 bit=1 位置对应的功能数据（队列发送）
3. **主机发送 `SYS_ASK_CMD`（CMD `0x03`）** — 请求单个 CMD 的功能值，VAL0 = 目标 CMD 编号

### 6.4 SYS_ASK_MAP 示例

主机发送 Map 查询（查询 byte 0-14 中所有 bit=1 的功能）：

```
主机发送: AA 01 0F 04 FF 00 3F 0F 9F 7F 80 FF FF 01 87 FF 00 8F BD
芯片回复: 55 01 02 ...（按 bit=1 的 CMD 逐一回复）
```

## 7. EQ 数据编码

### 7.1 dB 值与传输值映射

音乐 EQ 和 MIC EQ 的频段值在 -12 dB 到 +12 dB 之间显示，传输时线性映射为 0-24：

```
传输值 = 显示值 + 12
显示值 = 传输值 - 12
```

| 显示值（dB） | 传输值（byte） |
|-------------|---------------|
| -12 | 0 |
| -6 | 6 |
| 0 | 12 |
| +6 | 18 |
| +12 | 24 |

### 7.2 映射函数（JavaScript）

```javascript
// 显示值 → 传输值
function displayToWire(displayValue) {
    return displayValue + 12;
}

// 传输值 → 显示值
function wireToDisplay(wireValue) {
    return wireValue - 12;
}
```

### 7.3 10 段 EQ 帧组装示例

设置音乐 EQ：开关=开(1)，各频段为 -6, 0, +3, -12, 0, +6, 0, -3, +12, 0

```
显示值: [-6, 0, 3, -12, 0, 6, 0, -3, 12, 0]
传输值: [6, 12, 15, 0, 12, 18, 12, 9, 24, 12]

帧构建:
  Frame Header: 0xAA
  Web ID:       0x01
  LEN:          1 + 11 = 0x0C（CMD + 开关 + 10段）
  CMD:          0x15
  VAL0:         0x01（开关=开）
  VAL1-VAL10:   0x06, 0x0C, 0x0F, 0x00, 0x0C, 0x12, 0x0C, 0x09, 0x18, 0x0C
  CRC:          累加和 mod 256

完整帧: AA 01 0C 15 01 06 0C 0F 00 0C 12 0C 09 18 0C [CRC]
```

## 8. VAL 开关约定

多数音效、灯光、文字控制命令的第一个参数 VAL0 为开关控制：

| VAL0 值 | 含义 |
|---------|------|
| `0x00` | 关闭 / 禁用 |
| `0x01` | 开启 / 启用 |

其他参数（VAL1...VALn）仅在 VAL0=`0x01` 时有效。

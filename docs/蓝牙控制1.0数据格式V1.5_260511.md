## 1. 通用帧格式

所有 BLE 通信均采用定长/变长帧格式：

### Android → BLE（Web 发送）

| 帧头 | APP标识符 | 数据长度 len | 命令 cmd | 参数 n 字节 | CRC |
|------|-----------|-------------|---------|------------|-----|
| 0xAA | 0x01      | 1 字节      | 1 字节  | 0~n 字节   | 1 字节 |

- **帧头**: `0xAA` — Web/Android 发送标识
- **APP标识符**: `0x01`
- **数据长度 len**: 从命令字节开始到 CRC 之前的字节数
- **命令 cmd**: 操作指令
- **参数**: 变长，具体取决于命令
- **CRC**: 帧校验（从帧头到参数最后一字节的校验和，保留自定义算法）

### BLE → Android（芯片回复）

| 帧头 | 芯片标识符 | 数据长度 len | 命令 cmd | 参数 n 字节 | CRC |
|------|-----------|-------------|---------|------------|-----|
| 0x55 | 0x01      | 1 字节      | 1 字节  | 0~n 字节   | 1 字节 |

- **帧头**: `0x55` — 芯片回复标识
- **芯片标识符**: `0x01`

## 3. CMD 命令详解

### 3.1 Android → BLE 命令（帧头 0xAA）

| CMD | 名称 | 功能 | 参数 |
|-----|------|------|------|
| 0x00 | 问答机制 | 询问是否在线/连接 | CRC |
| 0x01 | 芯片支持功能表 | 查询芯片支持哪些功能 | — |
| 0xA0 | EQ 模式 + EQ1~8 值 | EQ 模式值(1~6) + EQ1~8 各值(0~24) | EQ模式(1字节) + EQ1值~EQ8值(各1字节) |
| 0xA9 | 音量 | 音量设置 | 值(0~40) |
| 0xAA | 低音 | 低音控制 | — |
| 0xAB | 高音 | 高音控制 | — |
| 0xB0 | 麦克风魔音效果 | 魔音模式选择 | 0x00关闭 / 0x01儿童 / 0x02女声 / 0x03男声 / 0x04电音 / 0x05魔音 |
| 0xB1 | 麦克风回声效果 | 回声开关 + 参数 | 0x00关闭 0x01打开 + 回声衰减(0~32) + 回声间隔(0~32) |
| 0xB2 | 麦克风混响效果 | 混响开关 + 参数 | 0x00关闭 0x01打开 + 混响大小(0~32) |
| 0xB3 | 麦克风防啸叫 | 防啸叫开关 | 0x00关闭 0x01打开 |
| 0xB4 | 麦克风频移 | 频移开关 + 参数 | 0x00关闭 0x01打开 + 值(0~5) |
| 0xB5 | 音乐消除人声 | 人声消除开关 + 参数 | 0x00关闭 0x01打开 + 值(0~100) |
| 0xB6 | 音乐 3D 丽音 | 3D 丽音开关 + 参数 | 0x00关闭 0x01打开 + 值(0~100) |
| 0xB7 | 音乐 虚拟低音 | 虚拟低音开关 + 参数 | 0x00关闭 0x01打开 + 值(0~100) |
| 0xEE | 一键恢复 | 恢复出厂设置 | CRC |
| 0xFB | 播放与暂停 | 播放/暂停控制 | 1=播放 / 0=暂停 + CRC |
| 0xFE | 文件信息 | OTA 文件传输信息 | 见文件传输协议 |
| 0xFF | 文件包数据传输 | OTA 数据包 | 见文件传输协议 |

### 3.2 芯片 → Android 回复（帧头 0x55）

| CMD | 名称 | 功能 | 参数 |
|-----|------|------|------|
| 0x00 | 应答机制 | 回复在线/连接确认 | CRC |
| 0x01 | 芯片支持功能表(动态长度) | bit 标识功能表 | 多字节 bit 标识(每字节 8 个功能) |
| 0xFD | 文件接收完成 | OTA 文件接收完毕确认 | 0x4F('O') + 0x4B('K') + CRC |
| 0xFE | get 文件信息 | 请求文件信息 | 文件名称(1字节) + CRC |
| 0xFF | get 文件包数据请求 | 请求指定包编号的数据 | 文件名称(1字节) + 包编号 3 字节(H/M/L) + CRC |

## 4. 音效详细协议格式

### 4.1 Android → BLE 音效指令

```
帧格式: 0xAA | 0x01 | len | cmd | [params] | CRC
```

#### EQ 模式 + 均衡器值（CMD 0xA0）

```
0xAA | len | 0xA0 | EQ_MODE | EQ1 | EQ2 | EQ3 | EQ4 | EQ5 | EQ6 | EQ7 | EQ8 | CRC
```

- **EQ_MODE**: 1~6
- **EQ1~EQ8**: 每个 0~24（映射 -12dB ~ +12dB）

#### 音量（CMD 0xA9）

```
0xAA | len | 0xA9 | value(0~40) | CRC
```

#### 低音（CMD 0xAA）

```
0xAA | len | 0xAA | CRC
```

#### 高音（CMD 0xAB）

```
0xAA | len | 0xAB | CRC
```

#### 麦克风魔音效果（CMD 0xB0）

```
0xAA | len | 0xB0 | mode | CRC
```

| mode | 效果 |
|------|------|
| 0x00 | 关闭 |
| 0x01 | 儿童 |
| 0x02 | 女声 |
| 0x03 | 男声 |
| 0x04 | 电音 |
| 0x05 | 魔音 |

#### 麦克风回声效果（CMD 0xB1）

```
0xAA | len | 0xB1 | on/off | decay(0~32) | interval(0~32) | CRC
```

- **decay**: 回声衰减
- **interval**: 回声间隔

#### 麦克风混响效果（CMD 0xB2）

```
0xAA | len | 0xB2 | on/off | size(0~32) | CRC
```

- **size**: 混响大小

#### 麦克风防啸叫（CMD 0xB3）

```
0xAA | len | 0xB3 | on/off | CRC
```

#### 麦克风频移（CMD 0xB4）

```
0xAA | len | 0xB4 | on/off | value(0~5) | CRC
```

#### 音乐消除人声（CMD 0xB5）

```
0xAA | len | 0xB5 | on/off | value(0~100) | CRC
```

#### 音乐 3D 丽音（CMD 0xB6）

```
0xAA | len | 0xB6 | on/off | value(0~100) | CRC
```

#### 音乐 虚拟低音（CMD 0xB7）

```
0xAA | len | 0xB7 | on/off | value(0~100) | CRC
```

#### 一键恢复（CMD 0xEE）

```
0xAA | 0x02 | 0xEE | CRC
```

#### 播放与暂停（CMD 0xFB）

```
0xAA | len | 0xFB | 1=播放 / 0=暂停 | CRC
```

## 5. 文件传输协议（OTA）

### 5.1 文件信息（CMD 0xFE）— 发送端 → 芯片

```
0xAA | 0x08 | 0xFE | file_name | file_len_H | file_len_M | file_len_L | file_crc_H | file_crc_L | CRC
```

| 字段 | 说明 |
|------|------|
| file_name | 文件名称编号 0~255（默认 0） |
| file_len_H/M/L | 文件总长度（3 字节，大端） |
| file_crc_H/L | 整个文件的 CRC 校验（2 字节） |
| CRC | 通讯协议自带帧校验 |

**示例**: `AA 08 FE 00 02 27 F0 6F 99 D1`

### 5.2 文件包数据传输（CMD 0xFF）

```
0xAA | len | 0xFF | pack_num_H | pack_num_M | pack_num_L | pack_data[1~140] | CRC
```

| 字段 | 说明 |
|------|------|
| pack_num_H/M/L | 包编号，从 0 开始计数（3 字节，大端） |
| pack_data | 包内容，单包长度 1~140 字节 |

**示例**: `AA FF 02 xx xx CRC`

### 5.3 芯片请求文件包（CMD 0xFF）— 芯片 → 发送端

```
0x55 | len | 0xFF | file_name | pack_num_H | pack_num_M | pack_num_L | CRC
```

| 字段 | 说明 |
|------|------|
| file_name | 文件名称 0~255 |
| pack_num_H/M/L | 请求的包编号，从 0 开始 |

### 5.4 文件信息请求（CMD 0xFE）— 芯片 → 发送端

```
0x55 | 0x02 | 0xFE | file_name | CRC
```

### 5.5 文件接收完成（CMD 0xFD）— 芯片 → 发送端

```
0x55 | len | 0xFD | 0x4F('O') | 0x4B('K') | CRC
```

## 6. 问答/枚举机制

### 初始连接流程

1. Web 发送 `0xAA 01 02 00 CRC`（SYS_ASK — 询问是否在线）
2. 芯片回复 `0x55 01 02 00 CRC`（应答确认）
3. Web 发送 `0xAA 01 02 01 CRC`（SYS_CHIP_SUPPORT_MAP — 询问功能表）
4. 芯片回复 `0x55 01 len 01 [bit map 多字节] CRC`（功能表列表）
5. Web 发送 `0xAA 01 02 04 CRC`（SYS_ASK_MAP — 请求获取当前值）
6. 芯片回复所有当前功能值

### 定时轮询

Web 按需发送 `SYS_ASK_CMD`（CMD 0x03）携带指定 CMD 编号，芯片回复对应功能的当前值。

| 方向 | 帧头 | 标识符 | 角色 |
|------|------|--------|------|
| Web → 芯片 | 0xAA | 0x01 (APP) | 控制指令、文件传输 |
| 芯片 → Web | 0x55 | 0x01 (芯片) | 应答、数据上报、文件请求 |

- 除 SYS_ASK / SYS_CHIP_SUPPORT_MAP / SYS_ASK_ALL / SYS_ASK_MAP / SYS_ASK_CMD 外，大部分音效/灯光/文字指令为 **Web 单向发送（W）**，芯片仅执行无需逐条回复。
- 文件传输（OTA）为双向交互，芯片通过不同 CMD 区分请求和回复。
